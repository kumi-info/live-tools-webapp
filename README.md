# LIVE Tools — Webアプリ版

**バージョン: v1.0.2**

TikTok LIVE の いいね／コイン／ギフト を OBS に表示し、視聴者アクションで演出や外部機器（Pavlok）を動かす、**exe不要・ブラウザだけで完結する**配信支援ツールです。

オーバーレイのURLは StreamToEarn のように短い固定URL（例 `http://localhost:21216/overlay/k7xq2m9a/coin`）で、**色や目標を変えてもURLを貼り直す必要はありません**。開いているOBSソースへ即時反映されます。

---

## できること

| 機能 | 説明 |
|---|---|
| 📊 メーター/カウンター | コイン・いいね・ギフト・ランキング・WIN の各オーバーレイ（OBS用） |
| 🎬 演出オーバーレイ | ＋WIN/−WIN/目標達成をトリガーに mp4/webm/gif/png/mp3 を再生 |
| 📈 配信分析 | 配信ごとの総いいね/コイン/フォローと視聴者別の内訳・視聴時間（CSV出力可） |
| ⚡ Pavlok連携 | `/pavlok/強さ_回数` で Pavlok デバイスを作動（安全上限あり） |

詳しい使い方はアプリ内の「❓ 使い方」を参照してください。

---

## 必要なもの
- Windows PC（OBSと同じPC）
- [Node.js LTS版](https://nodejs.org)（初回のみインストール）

## 使い方（かんたん）
1. `start.bat` をダブルクリック（初回は自動セットアップで1〜2分）
2. ブラウザで管理画面 `http://localhost:21216/` が開く
3. 「＋ 新しいセットを作成」→ 使いたいメーターの「📋 URL」をコピー
4. OBSの「ブラウザソース」に貼り付け（推奨 1920×1080）
5. 「⚙」で色・目標を設定 → OBSに自動反映
6. 管理画面の「接続」にTikTokユーザー名を入れて接続

> ポートを変えたいとき: `set PORT=21217 && node server.js`

## よくある質問・トラブル

- **設定はどこに保存される？** … このフォルダーの `data/` に保存されます。フォルダーごとコピーすれば設定も一緒に移動できます。
- **exe版と同時に使える？** … 起動は同時にできます（ポートが違うため）。ただし TikTok への接続はどちらか一方だけにしてください。
- **演出オーバーレイの音が出ない** … OBSのブラウザソースのプロパティで「OBSを介して音声を制御」を確認してください。
- **「ポートが使用中」で起動できない** … 前の起動ウィンドウを閉じてから起動し直してください（更新後は `start.bat` をもう一度実行。古いサーバーは自動で終了して入れ替わります）。
- **不具合が起きたとき** … 管理画面右下の「🐞 デバッグ / エラーログ」（`/debug`）で確認できます。配信に出るOBSのURLには表示されません。

## 仕組み（概要）

単一の Node.js プロセスが HTTP サーバーと Socket.IO サーバーを兼ね、`lib/*.js` の各機能を統括します。

```
TikTok LIVE ─► lib/tiktok.js ─┬─► io.emit（全オーバーレイへ配信）
                              └─► lib/analytics.js（配信データ）
OBS/ブラウザ ─► server.js（ルーティング＋Socket.IO）─► lib/profiles.js ─► data/*.json
              lib/pavlok.js ─► Pavlok Cloud API
```

| モジュール | 責務 |
|---|---|
| `server.js` | HTTPルーティング・Socket.IO・ページ/メディア配信・起動 |
| `lib/tiktok.js` | TikTok接続・イベント正規化・再接続・データ配信 |
| `lib/profiles.js` | オーバーレイ設定ストア（設定/WIN/状態の永続化） |
| `lib/analytics.js` | 配信データ集計（視聴者別・セッション復帰） |
| `lib/pavlok.js` / `lib/log.js` / `lib/followers.js` | Pavlok中継 / ログ集約 / リフォロー除外 |

- 設計ドキュメント: [要件定義書](docs/requirements.md) ／ [基本設計書](docs/basic-design.md) ／ [詳細設計書](docs/detailed-design.md)

## 設定・データの保存先

すべて `data/` に保存されます（フォルダーごとコピーすれば設定も移動できます）。

- `profiles.json` / `state.json` … 設定・WIN数・アプリ状態
- `broadcasts.json` … 配信データ（直近50配信）
- `pavlok.json` … Pavlok連携の設定

> ⚠ `data/` には視聴者データ・Pavlokトークンが含まれるため、`.gitignore` で公開対象から除外しています。**`data/` をそのまま公開・共有しないでください。**

## 開発・セットアップ

```bash
npm install          # 依存を取得（socket.io / tiktok-live-connector）
node server.js       # 起動（既定ポート 21216）
node --check server.js  # 構文チェック
```

- コーディング規約は Minecraft プラグイン群の憲章 `PLUGIN_PROJECT_RULES.md`（Google Style をベースに、コメント/JSDocは日本語）に準拠します。
- 配布時は `node_modules` と `data` を除いて zip 化してください（受け取り側の初回起動で自動生成されます）。
