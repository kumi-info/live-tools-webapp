**バージョン**: v1.0.2

**位置づけ**: TikTok LIVE 配信支援ツール「LIVE Tools」の **Webアプリ版**（exe不要・ブラウザ操作）の要件定義書。基本設計は [basic-design.md](basic-design.md)、詳細設計は [detailed-design.md](detailed-design.md) を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 名称 | LIVE Tools Webアプリ版（live-tools-webapp） |
| 種別 | ローカル常駐の Node.js HTTP ＋ Socket.IO サーバー |
| 目的 | TikTok LIVE の いいね／コイン／ギフト等を OBS に表示し、視聴者アクションで演出や外部機器（Pavlok）を動かす |
| 実行環境 | Windows PC（OBSと同じPC）＋ Node.js LTS |
| 既定ポート | 21216（`PORT` 環境変数で変更可。exe版 21214/21215 と同時起動可） |
| 配布形態 | フォルダーごと zip 配布（`node_modules`・`data` を除く）。受け取り側は `start.bat` 実行で自動セットアップ |
| データ保存 | すべて `data/`（配布時は自己完結・端末外に送信しない） |

## 2. 目的・背景

### 目的
- StreamToEarn（STE）等の**改変不可な既製ツールに依存せず**、配信演出・カウンターを自前で実装し、配信者がブラウザ設定だけで運用できる状態を作る。
- 従来の **exe版（Electron）** はインストール／更新が重いため、**ブラウザだけで完結する Web アプリ版**を用意する。

### 背景
- 用途は **TikTok 妨害配信**（視聴者のギフト／コメントで配信者を妨害・救済する参加型配信）の支援。
- 表示オーバーレイは STE のように**短い固定URL**（例 `/overlay/k7xq2m9a/coin`）で、設定を変えても **URL貼り直し不要**であることが強く求められる。
- WINカウンターは Minecraft プラグイン CommonWIN のオーバーレイをポーリングしてゲーム内WINを自動反映できる（役割分担。詳細は Minecraft 側の憲章 `PLUGIN_PROJECT_RULES.md` を参照）。累計WIN数・目標WIN数ともマイクラ側を正とし、マイクラ側の `/win reset`・サーバー再起動でURL側も0にそろえる。
- 累計WINは **アプリ（`start.bat`）を再起動するたびに0から**始める。OBS/ブラウザのリロードでは消さない（配信中に表示が飛ぶのを防ぐため）。

### 使用配信系統
- Ladder（はしご登り）／Farm（農場）／Building（建築）等、系統を問わず利用。

## 3. 機能要件

### 3.1 オーバーレイ（表示）機能

| 機能 | URL（`<id>`＝セットごとに自動発行） | 入力 | 出力 | 振る舞い |
|---|---|---|---|---|
| コインメーター | `/overlay/<id>/coin` | TikTokギフト | 累計コインのメーター | 目標達成でWIN加算・演出トリガー |
| いいねメーター | `/overlay/<id>/like` | いいね | 累計いいねのメーター | — |
| ギフトカウンター | `/overlay/<id>/gift` | ギフト | ギフト個数表示 | — |
| いいねランキング | `/overlay/<id>/ranking` | いいね | 視聴者別いいね順位 | — |
| コインランキング | `/overlay/<id>/coinrank` | ギフト | 視聴者別コイン順位 | — |
| WINカウンター | `/overlay/<id>/win` | 手動/自動 | 累計WIN数 | CommonWIN連携で自動加算・目標WIN数も追従・デザイン5種 |
| 演出オーバーレイ | `/overlay/<id>/alert` | 各トリガー | mp4/webm/gif/png/mp3の演出 | ＋WIN/−WIN/目標達成でルール発動 |
| URL一覧（人間用） | `/overlay/<id>` | — | そのセットの全URLコピー画面 | — |
| 設定編集 | `/edit/<id>/<type>` | 配信者 | 歯車UI | 変更は開いているOBSソースへ即時反映 |

- **設定はサーバー側（`data/profiles.json`）にID単位で保存**し、Socket.IO ルーム経由で該当オーバーレイへ即時反映する（URL不変）。

### 3.2 配信分析機能（`/analytics`）
- 配信ごとに 総いいね・総コイン・フォロー数、視聴者別のいいね・コイン・視聴時間（推定）を記録（`data/broadcasts.json`・直近50配信）。
- 取得 ON/OFF 可（OFFで集計・保存しない＝軽量・プライバシー）。CSV出力・タイトル修正・個別/全削除に対応。
- アプリが落ちても20分以内の再接続は同じ配信の続きとして復帰。

### 3.3 Pavlok連携機能（`/pavlok`）
- `GET/POST /pavlok/<強さ>_<回数>`（zap）等で Pavlok クラウドAPIへ中継。強さ100・回数5・送信間隔1.5秒の安全上限。まとめ投げはキューで順次発動。

### 3.4 運用支援機能
- 管理画面（`/`）・使い方（`/help`）・お知らせ（`/news`）・デバッグ/エラーログ（`/debug`）。
- 起動時、ポートを掴んだ**古い node サーバーだけ**を自動終了して入れ替わる。

## 4. 非機能要件

| 観点 | 要件 |
|---|---|
| **可用性（堅牢性）** | 参照ファイルの欠損・破損・別PCコピー時も既定値へフォールバックしクラッシュしない。`uncaughtException`/`unhandledRejection` を捕捉し `/debug` に記録して継続する。 |
| **保守性** | 機能ごとに `lib/*.js` へ分割（責務分離）。可変値は上限定数・設定ファイルに外出し。設計3点セット・README Changelog を実装と同時更新。 |
| **再利用性** | Pavlok・ログ等は独立モジュールとして他機能から再利用。exe版クライアント（HTML/JS）を無改変で使えるよう SYNC_MODE 文字列を一致させる。 |
| **可観測性** | 全モジュールが `lib/log.js` に集約ログ（source/level/detail・連発は集約）。主要イベントは `console.log`。`/debug` でブラウザから確認・コピー可。 |
| **セキュリティ／プライバシー** | 設定・発動系API（Pavlok等）は既定でループバック限定（`allowLan` で明示解放）。視聴者データ・トークン等の秘密情報は端末外に送信せず、公開リポジトリからは `.gitignore` で除外。パストラバーサル・ボディサイズ上限で入力を防御。 |
| **性能** | 高頻度イベント（1000人規模のいいね）は約150ms間隔でまとめて emit。設定保存は debounce＋アトミック書込（tmp→rename）。 |
| **配布容易性** | 環境非依存。データフォルダーは書込不可時に `%LOCALAPPDATA%` へ自動フォールバック。`start.bat` が Node 有無確認と `npm install` を自動化。 |

## 5. 制約・前提

- Windows 前提の機能あり（ホットキー送信＝`keybd_event`、ポート引き継ぎ＝PowerShell）。非Windowsでは該当機能を安全にスキップ。
- TikTok 接続は `tiktok-live-connector` に依存。ライブラリのバージョン差はフィールド正規化で吸収する。
- WINカウンターのマイクラ連携は CommonWIN プラグインのオーバーレイURL（既定 `http://localhost:8765`）をポーリングする。WINカウント本体等のゲーム内処理は Minecraft プラグイン側の責務。
